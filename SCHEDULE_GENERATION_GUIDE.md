# Schedule Generation Guide

## Порядок заполнения данных

Для корректной генерации расписания данные нужно заполнять в таком порядке:

1. `Teachers`
2. `Rooms`
3. `Groups`
4. `Disciplines`
5. `Sections`
6. Генерация расписания

## Что обязательно заполнить

### 1. Teachers
- `name`
- `email`
- `department`
- `teaching_languages`

### 2. Rooms
- `number`
- `capacity`
- `building`
- `type`
- `department`
- `equipment`
- `available`
- `computer_count`

### 3. Groups
- `name`
- `student_count`
- `has_subgroups`
- `language`

### 4. Disciplines
- `code`
- `name`
- `course`
- `semester`
- `programme`
- `department`
- `instructor_name`
- `requires_computers`

### 5. Sections
- `course_code`
- `group_name`
- `classes_count`
- `lesson_type`

## Логика генерации

`Discipline` описывает сам предмет.

`Section` описывает, для какой группы этот предмет проводится и сколько раз в неделю он должен стоять в расписании.

Без `sections` расписание не генерируется.

## Что должно совпадать

- у дисциплины должен быть назначен преподаватель
- язык группы должен входить в `teaching_languages` преподавателя
- вместимость аудитории должна быть не меньше числа студентов
- тип аудитории должен подходить типу занятия
- если у дисциплины `requires_computers = yes`, то аудитория должна иметь достаточный `computer_count`

## Соответствие типов

- `lecture` -> лекционная аудитория
- `practical` -> практическая аудитория
- `lab` -> лаборатория

## Частые причины ошибок

- не заполнены `sections`
- дисциплина есть, но преподаватель не назначен
- преподаватель не поддерживает язык группы
- не хватает аудиторий по вместимости
- для компьютерной дисциплины нет аудитории с достаточным числом ПК

## Рекомендуемый сценарий

1. Очистить старые данные при необходимости
2. Импортировать или заполнить `teachers`
3. Импортировать или заполнить `rooms`
4. Импортировать или заполнить `groups`
5. Импортировать или заполнить `disciplines`
6. Импортировать или заполнить `sections`
7. Выбрать нужный `semester` и `course`
8. Запустить генерацию
